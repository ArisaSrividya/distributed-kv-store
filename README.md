# Distributed Key-Value Store

## Overview

This project explores the design and implementation of a fault-tolerant distributed key-value store inspired by modern distributed databases.

The goal is to understand core distributed systems concepts including:

* Replication
* Consensus Algorithms
* Leader Election
* Write-Ahead Logging (WAL)
* Fault Tolerance
* Crash Recovery
* Consistency and Availability Trade-offs

## Planned Features

### Phase 1

* Single-node key-value store
* GET, SET, DELETE operations
* In-memory storage

### Phase 2

* Write-Ahead Logging (WAL)
* Crash recovery
* Persistent storage

### Phase 3

* Multi-node communication
* Replication

### Phase 4

* Raft consensus algorithm
* Leader election
* Log replication

### Phase 5

* Recovery and failover testing
* Performance benchmarking

## Technologies

* Java
* Distributed Systems Concepts
* Write-Ahead Logging
* Raft Consensus
